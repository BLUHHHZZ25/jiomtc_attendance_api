5) configure uvicorn + gunicorn + supervisor

- create the following files

sudo nano /etc/supervisord.d/iam-api.conf;
sudo mkdir /var/log/iam-api;
sudo nano /var/log/iam-api/iam-api.err.log;
sudo nano /var/log/iam-api/iam-api.out.log;

App creates it's own log file: /var/log/iam-api/iam-api.app.log - which is the main log file to check.

and edit the conf file iam-api.conf to look like something below:

sudo nano /etc/supervisord.d/iam-api.conf;

[program:iam-api]
directory=/home/ec2-user/iam-api/src
command=/home/ec2-user/iam-api/env/bin/gunicorn -c gunicorn_config.py main:app
autostart=true
autorestart=true
stderr_logfile=/var/log/iam-api/iam-api.err.log
stdout_logfile=/var/log/iam-api/iam-api.out.log


6) start the process via supervisor

sudo supervisorctl reread
sudo supervisorctl update

- make sure it is running by looking at

sudo supervisorctl status
sudo supervisorctl restart iam-api
netstat -tnlp

7) configure nginx

sudo nano /etc/nginx/conf.d/virtual.conf

and add the following location:


location  /iam-api/ {
    rewrite /iam-api/(.*) /$1 break;
    proxy_pass http://127.0.0.1:9000;
    proxy_set_header   Host             $host;
    proxy_set_header   X-Real-IP        $remote_addr;
    proxy_set_header   X-Forwarded-For  $proxy_add_x_forwarded_for;
    proxy_set_header   Upgrade $http_upgrade;
    proxy_set_header   Connection $http_connection;
    proxy_http_version 1.1;
}

sudo nano /etc/nginx/nginx.conf

    map $http_upgrade $connection_upgrade {
        default upgrade;
        '' close;
    }

8) verify and reload nginx:

sudo nginx -t && sudo service nginx restart
